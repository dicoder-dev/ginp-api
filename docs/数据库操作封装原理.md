# 数据库操作封装架构文档

## 概述

本项目采用分层架构实现数据库操作的高度封装，核心思想是将底层的 GORM 操作封装为业务层可直接调用的简洁 API，开发者无需编写原生 SQL 语句即可完成大部分数据库操作。

整体架构分为以下几个层次：

```
┌─────────────────────────────────────┐
│         Service 层 (业务层)          │  调用 Model 层方法执行业务逻辑
├─────────────────────────────────────┤
│         Model 层 (数据模型层)        │  封装实体类的 CRUD 操作
├─────────────────────────────────────┤
│         Entity/Field 层 (实体层)     │  定义数据库表结构和字段常量
├─────────────────────────────────────┤
│         dbops 包 (数据库操作封装)     │  提供通用的 CRUD 通用函数
├─────────────────────────────────────┤
│         where 包 (查询条件封装)       │  构建查询条件
├─────────────────────────────────────┤
│         dbs 包 (数据库初始化)         │  统一管理多数据库连接
└─────────────────────────────────────┘
```

---

## 一、数据库初始化层 (`internal/db`)

### 1.1 支持的数据库类型

项目支持三种数据库的初始化：

- **MySQL** (`internal/db/mysql`)
- **PostgreSQL** (`internal/db/pgsql`)
- **SQLite** (`internal/db/sqlite`)

### 1.2 统一入口 (`internal/db/dbs`)

通过 `dbs` 包提供统一的数据库获取入口，支持读写分离。

```go
// 获取读数据库实例
DbRead := dbs.GetReadDb()

// 获取写数据库实例
DbWrite := dbs.GetWriteDb()
```

数据库类型在配置文件中通过 `configs.Db.UseType` 指定，启动时自动初始化对应数据库。

## 二、查询条件层 (`pkg/where`)

### 2.1 Condition 结构体

用于描述单个查询条件，包含字段名、操作符、值和连接符。

```go
type Condition struct {
    Field   string      // 字段名
    Value   interface{} // 字段值
    Opt     string      // 操作符：= > < >= <= LIKE IN BETWEEN
    Connect string      // 连接符：AND / OR
}
```

### 2.2 支持的操作符

| 操作符 | 说明 | 示例值 |
|--------|------|--------|
| `=` | 等于 | 任意值 |
| `>` | 大于 | 任意值 |
| `<` | 小于 | 任意值 |
| `>=` | 大于等于 | 任意值 |
| `<=` | 小于等于 | 任意值 |
| `LIKE` | 模糊匹配 | `%keyword%` |
| `IN` | IN 查询 | `[]string{"a", "b"}` |
| `BETWEEN` | 范围查询 | `[]int{1, 10}` |

### 2.3 whereManager 链式构建

使用链式调用构建复杂查询条件：

```go
// 单条件
wheres := where.New("id", "=", 1).Conditions()

// 多条件（AND 连接）
wheres := where.New("status", "=", 1).
    And("username", "LIKE", "%admin%").
    Conditions()

// 多条件（OR 连接）
wheres := where.New("status", "=", 1).
    Or("status", "=", 2).
    Conditions()
```

### 2.4 Extra 额外参数

用于指定排序、分页等查询参数：

```go
// 创建分页查询参数
extra := &where.Extra{
    OrderByColumn: "created_at",  // 排序字段
    OrderByDesc:   true,           // 是否倒序
    PageSize:      10,             // 每页数量
    PageNum:       1,              // 页码
}

// 使用链式调用
extra := where.NewExtra().PSize(10).PNum(1).OrderBy("created_at", true)
```

## 三、数据库操作层 (`pkg/dbops`)

### 3.1 核心配置结构

#### FindOneConfig - 查询单条数据配置

```go
type FindOneConfig struct {
    Fields         []string              // 要查询的字段
    Wheres         []*where.Condition    // 查询条件
    NewEntity      any                    // 结果载体
    Db             *gorm.DB              // 数据库实例
    getSoftDelData bool                   // 是否查询软删除数据
    RelationList   []*RelationItem        // 关联查询配置
}
```

#### FindListConfig - 查询列表配置

```go
type FindListConfig struct {
    Conditions     []*where.Condition     // 查询条件
    Extra          *where.Extra           // 排序分页等
    NewEntityList  any                     // 结果载体
    GetSoftDelData bool                    // 是否查询软删除数据
    Db             *gorm.DB               // 数据库实例
    RelationList   []*RelationItem        // 关联查询配置
    Fields         []string                // 要查询的字段
}
```

#### DeleteConfig - 删除操作配置

```go
type DeleteConfig struct {
    NewEntity    any                       // 实体类型
    Wheres       []*where.Condition        // 删除条件
    SoftDelete   bool                      // 是否软删除
    Db           *gorm.DB                  // 数据库实例
    RelationList []string                  // 关联删除的字段
}
```

#### UpdateConfNew - 更新操作配置

```go
type UpdateConfNew struct {
    Db               *gorm.DB              // 数据库实例
    NewEntity        any                   // 实体类型
    DataUpdate       any                   // 更新数据
    Wheres           []*where.Condition    // 更新条件
    UpdateColumnsCfg []string               // 指定更新字段
}
```

### 3.2 更新字段配置常量

| 常量 | 说明 |
|------|------|
| `UpdateColumnsAll` | 强制更新所有字段（包括零值） |
| `UpdateColumnsNotZero` | 更新非零值字段 |
| `UpdateColumnsNotZeroAndStr` | 更新非零值字段 + 所有字符串类型字段 |
| `UpdateColumnsNotZeroAndNumber` | 更新非零值字段 + 所有数字类型字段 |

## 四、Model 层使用示例

### 4.1 Model 层定义

```go
type Model struct {
    dbWrite *gorm.DB
    dbRead  *gorm.DB
}

func NewModel(dbWrite_, dbRead_ *gorm.DB) *Model {
    return &Model{
        dbWrite: dbWrite_,
        dbRead:  dbRead_,
    }
}
```

### 4.2 CRUD 操作示例

#### 创建数据

```go
func (s *Model) Create(dtoCreate *entity.User) (*entity.User, error) {
    err := dbops.Create(dtoCreate, s.dbWrite)
    if err != nil {
        return nil, err
    }
    return dtoCreate, err
}
```

#### 查询单条

```go
func (s *Model) FindOne(wheres []*where.Condition) (*entity.User, error) {
    entityInfo := new(entity.User)
    err := dbops.FindOne(&dbops.FindOneConfig{
        Wheres:    wheres,
        Db:        s.dbRead,
        NewEntity: entityInfo,
    })
    if err != nil {
        return nil, err
    }
    return entityInfo, nil
}

// 根据 ID 查询
func (s *Model) FindOneById(id uint) (*entity.User, error) {
    return s.FindOne(where.New(FieldID, "=", id).Conditions())
}
```

#### 查询列表

```go
func (s *Model) FindList(wheres []*where.Condition, extra *where.Extra) ([]*entity.User, uint, error) {
    var entityList []*entity.User
    err := dbops.FindList(&dbops.FindListConfig{
        Conditions:     wheres,
        Db:             s.dbRead,
        Extra:          extra,
        NewEntityList:  &entityList,
        GetSoftDelData: false,
    })
    if err != nil {
        return nil, 0, err
    }

    // 统计总数
    total, err := dbops.GetTotal(wheres, new(entity.User), s.dbRead)
    if err != nil {
        return []*entity.User{}, 0, err
    }

    return entityList, uint(total), nil
}
```

#### 更新数据

```go
func (s *Model) Update(wheres []*where.Condition, dtoUpdate *entity.User, columnsCfg ...string) error {
    err := dbops.Update(&dbops.UpdateConfNew{
        Wheres:           wheres,
        NewEntity:        new(entity.User),
        Db:               s.dbWrite,
        UpdateColumnsCfg: columnsCfg,
        DataUpdate:       dtoUpdate,
    })
    return err
}
```

#### 删除数据

```go
func (s *Model) Delete(wheres []*where.Condition) error {
    err := dbops.Delete(&dbops.DeleteConfig{
        Wheres:     wheres,
        Db:         s.dbWrite,
        SoftDelete: false,
        NewEntity:  new(entity.User),
    })
    return err
}

func (s *Model) DeleteById(id uint) error {
    return s.Delete(where.New(FieldID, "=", id).Conditions())
}
```

## 五、Service 层调用示例

```go
// 查询用户
user, err := Model().FindOne(where.New(muser.FieldID, "=", userID).Conditions())

// 分页查询用户列表
list, total, err := Model().FindList(
    where.New(muser.FieldStatus, "=", 1).Conditions(),
    where.NewExtra().PSize(10).PNum(1).OrderBy("created_at", true),

// 更新用户信息
err = Model().Update(
    where.New(muser.FieldID, "=", user.ID).Conditions(),
    userEntity,

// 删除用户
```

## 七、特性总结

1. **多数据库支持**：通过配置文件可灵活切换 MySQL、PostgreSQL、SQLite
2. **读写分离**：提供独立的读库和写库实例
3. **链式条件构建**：使用 `whereManager` 优雅构建复杂查询条件
4. **操作符丰富**：支持 =、>、<、>=、<=、LIKE、IN、BETWEEN 等
5. **分页支持**：内置分页参数构建和偏移量计算
6. **关联查询**：支持预加载关联数据
7. **软删除**：支持查询和删除软删除的数据
8. **零值更新**：提供多种更新策略解决 Go 零值问题
9. **字段常量**：自动生成实体字段常量，避免硬编码字符串
