# ginp-api 项目架构文档

## 一、项目概述

ginp-api 是一个基于 Go + Gin 框架开发的 RESTful API 项目，采用分层架构设计，包含完整的代码生成工具，支持多数据库（MySQL、PostgreSQL、SQLite）。

## 三、命名规范

### 1. 文件命名规范

| 层级 | 后缀 | 说明 | 示例 |
|------|------|------|------|
| Controller | `.a.go` | Action处理器 | `sys_user_create.a.go` |
| Service | `.s.go` | Service层 | `user.s.go`、`cache.s.go` |
| Model | `.m.go` | 数据模型 | `user.m.go` |
| Entity | `.e.go` | 实体定义 | `user.e.go` |
| Field | `.f.go` | 字段常量 | `user.f.go` |
| DTO | `.go` | 数据传输对象 | `delete.go`、`find.go` |
| Router | `.go` | 路由配置 | `router.go`、`use_*.go` |
| Start | `.go` | 启动配置 | `start_gin.go` |
| Helper | `.go` | 辅助函数 | `email.go` |

### 2. 包命名规范

```
controller/模块名/包名/
  - cuser    # c + user（控制器用户包）
  - ccommon  # c + common（公共控制器）
  - cindex   # c + index（索引控制器）

service/模块名/包名/
  - suser    # s + user（服务用户包）
  - scache   # s + cache（缓存服务）

model/模块名/包名/
  - muser    # m + user（模型用户包）

entity/
  - 直接使用实体名，如 user, test, demo_table
```

### 3. 变量命名规范

- **Model结构体**: `Model` (如 `type Model struct`)
- **Service单例**: 包名大写 (如 `var User *muser.Model`)
- **字段常量**: `Field` + 驼峰命名 (如 `FieldID`, `FieldUsername`)
- **表名常量**: `tableName` + 驼峰 (如 `tableNameUser`)

### 4. 路由命名规范

- RESTful风格
- 路径使用下划线命名
- 模块名前缀（如 `sys_`, `api/`）

示例：
```
/api/sys_user/create      # 创建
/api/sys_user/delete      # 删除
/api/sys_user/update      # 更新
/api/sys_user/find_by_id  # 按ID查询
/api/sys_user/search      # 搜索列表
/api/user/login           # 登录
/api/user/register       # 注册
```

### 5. 代码生成文件后缀说明

| 后缀 | 含义 | 来源 |
|------|------|------|
| `.a.go` | Action/Controller | 自动生成 |
| `.s.go` | Service | 自动生成 |
| `.m.go` | Model | 自动生成 |
| `.e.go` | Entity | 自动生成 |
| `.f.go` | Fields（字段常量） | 自动生成 |
| `.r.go` | Router | 自动生成 |

## 四、架构思想

### 1. 分层架构

```
┌─────────────────────────────────────────┐
│           Controller 层                  │
│     (HTTP 请求处理、参数绑定、响应返回)     │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│            Service 层                     │
│        (业务逻辑处理、事务控制)             │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│             Model 层                      │
│      (数据库操作、CRUD、查询构建)           │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│            Entity 层                      │
│         (数据模型定义、表映射)              │
└─────────────────────────────────────────┘
```

### 2. 代码生成核心思想

项目实现了自动代码生成工具 `gencode`，核心流程：

1. **定义实体** → 在 `entity/` 目录定义数据模型
2. **实现接口** → 实体实现 `typ.IEntity` 接口
3. **运行生成** → 执行 `go run cmd/gencode/main.go` 命令
4. **自动生成** → 自动生成 Controller、Service、Model、Field 常量等代码

### 3. 数据库访问模式

- **读写分离**: Model 层支持 `dbRead` 和 `dbWrite` 两个数据库实例
- **条件构建**: 使用 `pkg/where` 包构建查询条件
- **软删除**: 支持软删除机制

### 4. 中间件机制

- CORS 跨域支持
- 登录鉴权（可选）
- 权限验证（可选）
- 请求日志

### 5. 扩展性设计

- **多数据库支持**: MySQL、PostgreSQL、SQLite
- **云存储集成**: 腾讯云 COS
- **邮件服务**: 支持验证码发送
- **任务调度**: 定时任务支持

### 6. 启动流程

```
start.Run()
  ├── startDB()        # 初始化数据库连接
  ├── startTask()      # 启动定时任务
  ├── startGinLogger() # 配置日志
  └── startGinServer() # 启动HTTP服务
```

## 五、关键技术栈

- **Web框架**: Gin
- **ORM**: GORM
- **数据库**: MySQL / PostgreSQL / SQLite
- **配置**: YAML
- **日志**: 自定义日志库
- **代码生成**: 自研 gencode 工具

## 六、开发流程

1. **创建实体**: 在 `internal/gapi/entity/` 创建 `.e.go` 文件
2. **运行生成**: 执行 gencode 命令生成完整代码
3. **实现业务**: 在生成的 Controller 中编写业务逻辑
4. **配置路由**: 自动注册路由，无需手动配置
