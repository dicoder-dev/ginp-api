# Go 项目标准目录结构

## 整体结构

```
myproject/
├── cmd/
│   └── myapp/       // 主应用程序入口
│       └── main.go  // 应用程序的main函数
├── internal/        // 私有应用和库代码
│   ├── app/         // 应用层逻辑
│   ├── domain/      // 领域模型
│   └── pkgname/     // 更多内部包...
├── pkg/             // 可以被外部使用的库代码
│   └── pkgname/     // 包名
├── api/             // API定义（例如：OpenAPI、gRPC）
├── configs/         // 配置文件
├── scripts/         // 构建、部署、数据库迁移等脚本
├── build/           // 构建输出目录
├── docs/            // 文档
├── web/             // 前端文件或者静态web内容
├── test/            // 测试相关的资源或脚本
├── vendor/          // Go modules依赖包（如果使用go mod vendor）
├── .github/         // GitHub workflows, issue templates 等
├── .gitignore       // Git忽略文件配置
├── go.mod           // Go module文件
└── go.sum           // Go checksum database
```

## cmd 目录结构

```
cmd/
├── myapp/
│   ├── main.go        // 主应用程序入口
│   └── other_main.go  // 其他启动点（如果适用）
├── mytool/
│   └── main.go        // 管理工具或其他辅助工具入口
```

> 如果你的项目需要生成多个二进制文件（例如，一个后台服务、一个管理工具等），你可以在cmd/目录下为每个应用创建一个子目录。每个子目录下的main.go文件将是该应用的入口点。

## 本项目 ginp-api 的目录结构

```
internal/
├── gapi/                      // 当前应用模块
│   ├── controller/            // 控制器层：处理HTTP请求与响应
│   │   ├── user/cuser/        // 用户相关控制器
│   │   └── system/            // 系统相关控制器
│   ├── service/              // 业务逻辑层
│   │   ├── user/suser/        // 用户服务
│   │   └── system/            // 系统服务
│   ├── dto/                  // 数据传输对象
│   │   └── comdto/           // 通用DTO
│   ├── entity/               // 实体类（与数据库表映射）
│   │   ├── user.e.go
│   │   └── demo_table.e.go
│   ├── model/                // 数据模型
│   │   ├── user/muser/
│   │   └── system/mdemotable/
│   ├── router/               // 路由配置
│   │   ├── router.go
│   │   └── use_*.go          // 中间件
│   └── start/                // 启动配置
│       ├── run.go
│       ├── start_gin.go
│       ├── start_db.go
│       └── start_task.go
│
└── db/                       // 数据库相关逻辑
    ├── mysql/                // MySQL 初始化
    ├── pgsql/                // PostgreSQL 初始化
    ├── sqlite/               // SQLite 初始化
    └── dbs/                  // 数据库实例管理
```

## pkg 公共库

```
pkg/
├── ginp/                     // Gin 框架扩展 (ContextPlus)
├── dbops/                   // 数据库操作工具
├── where/                   // 查询条件构建
├── utils/                   // 通用工具函数
├── cos/                     // 腾讯云 COS
├── email/                   // 邮件发送
├── filehelper/              // 文件处理
├── logger/                  // 日志
├── httpclient/              // HTTP 客户端
├── task/                    // 任务调度
├── upload/                  // 文件上传
├── cfg/                     // 配置管理
└── maps/                    // Map 操作工具
```

## 控制器代码文件命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 控制器 | `xxx.a.go` | `user_search.a.go` |
| 服务 | `xxx.s.go` | `user.s.go` |
| 模型 | `xxx.m.go` | `user.m.go` |
| 实体 | `xxx.e.go` | `user.e.go` |
| 路由 | `xxx.r.go` | `index.r.go` |
| 字段常量 | `xxx.f.go` | `user.f.go` |
