# 配置文件

## 概述

项目使用结构体 + Tag 的方式管理配置，支持自动生成配置键、多级嵌套、热加载和同步写入。

## 核心文件

- `pkg/cfg/helper.go` - 配置解析核心逻辑
- `configs/` - 各模块配置文件

## 配置结构体定义

### 基本结构

```go
type ServerConfig struct {
    Port string `default:"8082"`
}
```

- **Tag 说明**：
  - `default`: 配置默认值
  - `configkey`: 可选，手动指定配置键，不指定则自动生成

### 自动生成规则

- 结构体名 + 字段名转小写下划线
- 自动去除 `Config` 后缀
- 例如：`ServerConfig.Port` → `server.port`
- 例如：`Server.Port` → `server.port`（兼容处理）
- 例如：`EmailConfig.Client.Account` → `email.client.account`

### 多级嵌套

```go
type ClientConfig struct {
    Account string `default:"test@126.com"`
    Pwd     string `default:"123456"`
    Port    int    `default:"465"`
}

type EmailConfig struct {
    Client ClientConfig
}
```

生成的配置键：
- `email_config.client.account`
- `email_config.client.pwd`
- `email_config.client.port`

## 使用方式

### 1. 定义配置结构体

```go
// configs/email.go
package configs

import "ginp-api/pkg/cfg"

type ClientConfig struct {
    Account string `default:"dicoder@126.com"`
    Pwd     string `default:"12345"`
    Port    int    `default:"465"`
    Host    string `default:"smtp.126.com"`
}

type EmailConfig struct {
    Client ClientConfig
}

var Email = new(EmailConfig)

func init() {
    cfg.ParseConfigStruct(Email)
}
```

### 2. 读取配置

```go
import "ginp-api/configs"

// 访问嵌套字段
host := configs.Email.Client.Host
port := configs.Email.Client.Port

// 访问顶层字段
useType := configs.Db.UseType
```

### 3. 写入配置

#### 方式一：设置单个值

```go
import "ginp-api/pkg/cfg"

// 设置值并写入配置文件
cfg.Set("server.port", "9000")
```

#### 方式二：直接赋值后同步

```go
import (
    "ginp-api/configs"
    "ginp-api/pkg/cfg"
)

// 修改配置值
configs.Email.Client.Port = 587

// 同步到配置文件
cfg.SyncConfig(Email)
```

### 4. 热加载

当配置文件被外部修改后，可以调用 Reload 重新加载：

```go
import "ginp-api/pkg/cfg"

// 重新加载配置文件
err := cfg.Reload()
if err != nil {
    panic(err)
}
```

## API 参考

### pkg/cfg

| 函数 | 说明 |
|------|------|
| `ParseConfigStruct(ptr)` | 解析配置结构体，设置默认值并读取配置值 |
| `Get(key)` | 获取配置值（any 类型） |
| `GetString(key)` | 获取字符串配置 |
| `GetInt(key)` | 获取整数配置 |
| `GetBool(key)` | 获取布尔配置 |
| `GetFloat(key)` | 获取浮点数配置 |
| `Set(key, value)` | 设置配置值（写入配置文件） |
| `SetDefault(key, value)` | 设置默认值（仅在值不存在时生效） |
| `SyncConfig(ptr)` | 将结构体的值同步到配置文件 |
| `Reload()` | 重新加载配置文件 |

## 注意事项

1. **init 顺序**：配置文件的 init 会在 main 之前执行，确保配置已加载
2. **默认值优先级**：`default` tag 只在配置文件中没有对应值时生效
3. **嵌套结构**：嵌套的结构体不需要额外调用 `ParseConfigStruct`，会自动递归解析
4. **热加载后需重新初始化**：Reload 后，某些依赖配置的组件可能需要重新初始化
