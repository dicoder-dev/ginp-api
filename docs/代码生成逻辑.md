# 代码生成工具文档

## 概述

本文档介绍 GAPI 代码生成工具的设计思路和实现逻辑。该工具用于自动生成 GINP-API 项目中的 CRUD 代码、API 接口、字段常量等。

## 交互式菜单

入口文件：`cmd/gencode/cmd/interactive.go`

### 主菜单

```
=== GAPI 代码生成工具 ===
请选择操作：
1. 生成实体 CRUD 代码
2. 新增 API 接口控制器
3. 生成实体字段常量
4. 删除实体 CRUD 代码
5. 退出
```

### 菜单选项说明

| 编号 | 功能 | 说明 |
|------|------|------|
| 1 | 生成实体 CRUD 代码 | 为实体生成完整的增删改查代码 |
| 2 | 新增 API 接口控制器 | 在已有控制器目录下新增自定义 API |
| 3 | 生成实体字段常量 | 扫描所有实体并生成字段常量文件 |
| 4 | 删除实体 CRUD 代码 | 删除指定实体的所有 CRUD 相关文件 |

### 通用交互逻辑

- **回车默认选择**：所有选项选择时，直接按回车会默认选择第一个选项
- **提示信息**：每个选项旁边会显示 "（直接回车默认选择 X）" 提示用户
- **按回车继续**：操作完成后显示 "按回车键继续..."

## 1. CRUD 代码生成

### 1.1 生成的文件

为一个实体生成 CRUD 代码时，会创建以下文件：

| 文件类型 | 路径格式 | 命名示例 |
|----------|----------|----------|
| 实体文件 | `entity/{lineName}.e.go` | `user_data.e.go` |
| 控制器文件 | `controller/[parent/]c{lineName}/*.a.go` | `cuserdata/*.a.go` (5个) |
| 服务文件 | `service/[parent/]s{lineName}/{lineName}.s.go` | `suserdata/user_data.s.go` |
| 模型文件 | `model/[parent/]m{lineName}/{lineName}.m.go` | `muserdata/user_data.m.go` |
| 路由文件 | `controller/[parent/]c{lineName}/{lineName}.r.go` | `cuserdata/user_data.r.go` |

### 1.2 控制器文件列表

CRUD 生成器会为每个实体生成 5 个控制器文件：

| 文件 | 功能 |
|------|------|
| `{entity}_create.a.go` | 创建记录 |
| `{entity}_find_by_id.a.go` | 根据 ID 查询 |
| `{entity}_update.a.go` | 更新记录 |
| `{entity}_delete.a.go` | 删除记录 |
| `{entity}_search.a.go` | 条件查询列表 |

### 1.3 目录结构

```
internal/gapi/
├── entity/                          # 实体定义
│   └── user_data.e.go              # 实体文件
├── controller/                      # 控制器层
│   ├── system/                      # system 父级目录
│   │   └── cuserdata/              # 控制器目录
│   │       ├── user_data.a.go     # 控制器
│   │       └── user_data.r.go     # 路由
│   └── cuser/                      # 顶层控制器（无父级）
├── service/                        # 服务层
│   ├── system/
│   │   └── suserdata/
│   │       └── user_data.s.go
│   └── suser/
├── model/                          # 模型层
│   ├── system/
│   │   └── muserdata/
│   │       ├── user_data.m.go
│   │       └── user_data.f.go     # 字段常量
│   └── muser/
└── router/
    └── router.go                   # 主路由文件
```

### 1.4 实体配置

实体文件包含 `GenConfig()` 方法，返回实体配置：

```go
func (UserData) GenConfig() *gen.EntityConfig {
    return &gen.EntityConfig{
        TableName:        "user_data",      // 数据库表名
        FatherFolderName: "system",         // 父级目录名称（可选）
    }
}
```

### 1.5 关键函数

| 函数 | 文件 | 说明 |
|------|------|------|
| `GenBatchCrudWithParent` | `desc/gen_crud.go` | 批量生成 CRUD，支持父级目录 |
| `genCrud` | `desc/gen_crud.go` | 单个实体 CRUD 生成核心逻辑 |
| `getBaseReplaceMap` | `desc/constants.go` | 获取基础替换变量 |
| `PathEntity` | `desc/path_format.go` | 实体文件路径 |
| `PathController` | `desc/path_format.go` | 控制器目录路径 |

## 2. 字段常量生成

### 2.1 生成逻辑

入口函数：`desc/gen_fields.go` 中的 `GenFields()`

#### 流程

1. **扫描实体文件**：扫描 `entity/` 目录下所有 `.e.go` 文件
2. **解析字段**：使用正则表达式解析 struct 中的字段名
3. **检测父级目录**：从实体的 `FatherFolderName` 配置获取
4. **生成常量**：为每个字段生成对应的常量定义

#### 字段处理规则

| 字段类型 | 常量命名 | 值 |
|----------|----------|-----|
| 普通字段 | `Field{FieldName}` | 转为下划线命名 |
| ID | `FieldID` | `"id"` |
| CreatedAt | `FieldCreatedAt` | `"created_at"` |
| UpdatedAt | `FieldUpdatedAt` | `"updated_at"` |
| DeletedAt | `FieldDeletedAt` | `"deleted_at"` |

#### 示例

实体定义：
```go
type UserData struct {
    ID        uint      `json:"id"`
    Name      string    `json:"name"`
    CreatedAt time.Time `json:"created_at"`
    UpdatedAt time.Time `json:"updated_at"`
}
```

生成的字段常量：
```go
package muserdata

const FieldID = "id"
const FieldName = "name"
const FieldCreatedAt = "created_at"
const FieldUpdatedAt = "updated_at"
```

### 2.2 关键函数

| 函数 | 说明 |
|------|------|
| `ScanExistingEntities` | 扫描所有实体文件 |
| `genFieldsForEntity` | 为单个实体生成字段常量 |
| `parseEntityFields` | 解析实体文件中的字段名 |
| `detectEntityParentDirForFields` | 检测实体父级目录 |
| `PathFields` | 生成字段常量文件路径 |

### 2.3 路径格式

```
model/{parentDir}/m{lineName}/{lineName}.f.go
```

示例：
- 无父级目录：`model/muser/user.f.go`
- 有父级目录：`model/system/muserdata/user_data.f.go`

## 3. API 接口生成

### 3.1 生成逻辑

用户通过交互式菜单选择：
1. 选择目标目录（父级目录）
2. 选择或新建子目录（必须以 `c` 开头）
3. 输入 API 名称（大驼峰命名）

### 3.2 API 文件结构

```
controller/{parentDir}/c{apiName}/
├── {apiName}.a.go      # 控制器实现
└── {apiName}.r.go     # 路由定义
```

### 3.3 关键函数

| 函数 | 说明 |
|------|------|
| `ScanControllerDirs` | 扫描所有控制器父级目录 |
| `ScanSubDirs` | 扫描父级目录下的子目录 |
| `GenAddApiWithParams` | 生成 API 文件 |

## 4. CRUD 删除功能

### 4.1 删除逻辑

入口函数：`desc/remove_batch_crud.go` 中的 `RemoveBatchCrudWithParent()`

#### 删除流程

1. 遍历要删除的实体列表
2. 对每个实体执行以下删除操作：
   - 删除实体文件 (`*.e.go`)
   - 删除路由文件 (`*.r.go`)
   - 删除控制器目录下所有 `.a.go` 文件
   - 删除服务文件 (`*.s.go`)
   - 删除模型文件 (`*.m.go`)
   - 删除字段常量文件 (`*.f.go`)
   - 清理空目录
   - 移除路由导入

### 4.2 父级目录检测

删除时需要知道实体的父级目录，检测逻辑：

1. **优先从实体配置获取**：读取实体文件中的 `FatherFolderName` 字段
2. **fallback 扫描目录**：遍历 controller 目录查找匹配的实体目录

### 4.3 关键函数

| 函数 | 说明 |
|------|------|
| `RemoveBatchCrudWithParent` | 批量删除 CRUD 代码 |
| `removeControllerFiles` | 删除控制器目录下的 .a.go 文件 |
| `removeEmptyDirectories` | 清理空目录 |
| `detectEntityParentDir` | 检测实体父级目录 |
| `getEntityFatherFolderName` | 从实体文件获取父级目录配置 |

## 5. 模板系统

### 5.1 模板文件位置

`cmd/gencode/templates/`

### 5.2 可用变量

| 变量 | 说明 | 示例 |
|------|------|------|
| `$ENTITY_NAME$` | 实体名称（大驼峰） | `UserData` |
| `$ENTITY_LINE$` | 实体名称（下划线） | `user_data` |
| `$PACKAGE_NAME$` | 包名（全小写） | `userdata` |
| `$API_NAME_BIG$` | API 名称（大驼峰） | `GetUserInfo` |
| `$API_NAME_LINE$` | API 名称（下划线） | `get_user_info` |
| `$PARENT_DIR$` | 父级目录（带斜杠） | `system/` |
| `$PARENT_DIR_PREFIX$` | 父级目录（带斜杠前缀） | `/system` |
| `$FATHER_FOLDER_NAME$` | 父级目录名称 | `system` |
| `$ENTITY_TITLE$` | 实体标题 | `UserData` |

### 5.3 模板列表

| 模板文件 | 用途 |
|----------|------|
| `entity.tmpl` | 实体文件模板 |
| `controller_create.tmpl` | 创建控制器模板 |
| `controller_find_by_id.tmpl` | 查询单个控制器模板 |
| `controller_update.tmpl` | 更新控制器模板 |
| `controller_delete.tmpl` | 删除控制器模板 |
| `controller_search.tmpl` | 搜索控制器模板 |
| `service.tmpl` | 服务层模板 |
| `model.tmpl` | 模型层模板 |
| `router.tmpl` | 路由模板 |
| `fields.tmpl` | 字段常量模板（已弃用，改用代码生成） |

## 9. 常见问题

### Q1: 生成字段常量时提示 "no such file or directory"

**原因**：model 目录不存在（实体没有生成过 CRUD）

**解决**：先为实体生成 CRUD 代码，再生成字段常量

### Q2: 删除 CRUD 时文件未删除

**原因**：父级目录检测失败

**解决**：确保实体文件中有 `FatherFolderName` 配置

### Q3: API 文件生成位置错误

**原因**：子目录命名未以 `c` 开头

**解决**：新建子目录时必须以 `c` 开头
